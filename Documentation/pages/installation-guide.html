Batocera-Specific Features

  • Auto-Resolution: Automatically detects and uses your screen's native resolution
  • Controller Support: Works with any controller configured in Batocera
  • Save Data: Saves are stored in the game directory and persist across reboots
  • Low Resource Usage: Optimized for handheld devices and low-power systems

Troubleshooting Batocera

Game doesn't appear in EmulationStation:
  • Ensure the omnipet.pygame file is in the correct location
  • Check that the file has execute permissions
  • Restart EmulationStation or reboot Batocera
Game crashes on startup:
  • Check that all game files are present
  • Verify Python syntax in the launcher file
  • Look for error messages in /tmp/pygame_stdout.log

Post-Installation Setup

First Launch

When you first start Omnipet:

  1. Make sure you have a config directory with the config files in it
  2. To access the module documentation from within the game main documentation files, run the game at least once
  3. You'll see the egg selection screen, with eggs available from all installed modules
  4. You'll see the main game screen with an egg ready to hatch

Configuration

Key configuration files:

  • config.json: Default/Fallback display settings, sound levels, and game options
  • input_config.json: Default/Fallback button mappings

  • config_batocera.json: Display settings, sound levels, and game options for Batocera
  • input_config_batocera.json: Customize button mappings for Batocera

  • config_python_desktop.json: Display settings, sound levels, and game options for Linux or macOS
  • input_config_python_desktop.json: Customize button mappings for Linux or macOS

  • config_raspberry.json: Display settings, sound levels, and game options for Raspberry Pi OS
  • input_config_raspberry.json: Customize button mappings for Raspberry Pi OS

  • config_windows.json: Display settings, sound levels, and game options for Windows
  • input_config_windows.json: Customize button mappings for Windows

  • Module configs: Each module has its own configuration options, configured by editing the .json files in the module folder, or using Module Editor on Windows

Adding Modules

To add new modules:

  1. Create a new folder in the modules/ directory
  2. Add the required module files (module.json, monster.json, etc.)
  3. Restart the game to load the new module
  4. The module documentation will be added to the guide list
